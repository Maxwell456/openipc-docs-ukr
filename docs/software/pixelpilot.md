# PixelPilot — наземна станція на Android

**PixelPilot** — безкоштовний open source додаток, що перетворює Android-смартфон чи планшет на повноцінну наземну станцію OpenIPC. Це найдешевший спосіб почати літати: не потрібні ні окуляри, ні одноплатний комп'ютер — достатньо телефона та Wi-Fi адаптера за ~$10–15.

Що вміє PixelPilot:

- Приймати відео **WFB-NG** через зовнішній Wi-Fi адаптер (OTG)
- Відтворювати **RTP/UDP-потік** з прошивки [APFPV](/software/apfpv) — без адаптера, напряму через Wi-Fi мережу дрона
- Декодувати **H.264 та H.265**
- Показувати **OSD-телеметрію** (MSP з польотного контролера)
- Записувати **DVR** у пам'ять телефона

::: info Дві версії PixelPilot
- **[PixelPilot](https://github.com/OpenIPC/PixelPilot/releases)** — для Android-смартфонів і планшетів (ця сторінка)
- **[PixelPilot_rk](https://github.com/OpenIPC/PixelPilot_rk)** — для одноплатних комп'ютерів на Rockchip (Radxa Zero 3W та ін.) з виводом на HDMI; використовується в [наземних станціях на Radxa](/getting-started/groundstation-build)
:::

## Що потрібно

| Компонент | Вимоги |
|---|---|
| Android-пристрій | Підтримка USB OTG; що потужніший процесор — то менша затримка |
| Wi-Fi адаптер | [RTL8812AU](/hardware/net-cards/rtl8812au) (рекомендовано) або [RTL8812EU2](/hardware/net-cards/rtl8812eu) з живленим USB-хабом |
| OTG-кабель/адаптер | USB-C або micro-USB → USB-A |
| Файл `gs.key` | Ключ шифрування з камери (як його отримати — [інструкція для WiFiLink v2](/hardware/vtx/runcamwifilinkv2)) |

Для режиму APFPV адаптер і `gs.key` не потрібні — телефон просто підключається до Wi-Fi мережі дрона.

## Встановлення

1. Завантажте останній APK зі сторінки [релізів PixelPilot](https://github.com/OpenIPC/PixelPilot/releases)
2. Дозвольте встановлення з невідомих джерел (Android запитає автоматично)
3. Встановіть додаток і запустіть його

## Налаштування для WFB-NG

1. Скопіюйте файл `gs.key` з камери на телефон
2. Підключіть Wi-Fi адаптер через OTG — Android запитає дозвіл на доступ до USB-пристрою, підтвердьте та відкрийте PixelPilot через спливаюче вікно
3. Натисніть іконку шестерні та встановіть:
   - **Channel** — `161` (має збігатися з каналом камери)
   - **Bandwidth** — `20` МГц
4. Додайте ключ: **Gear → WFB-NG → gs.key** → виберіть скопійований файл
5. Увімкніть VTX — відео з'явиться протягом кількох секунд

::: tip Канал і ширина каналу
`161 / 20 МГц` — типові значення для стокових прошивок Runcam WiFiLink. Якщо ви змінювали канал на камері (через [VTX Menu](/configuration/telemetry) чи конфіг), виставте в PixelPilot ті самі значення.
:::

## Налаштування для APFPV

1. Увімкніть дрон із прошивкою [APFPV](/software/apfpv) — він створить Wi-Fi мережу
2. Підключіть телефон до цієї мережі
3. Відкрийте PixelPilot — відео з'явиться автоматично

## Запис DVR

PixelPilot пише відео у пам'ять телефона у форматі MP4. Кнопка запису — на головному екрані під час польоту. Обсяг файлів залежить від бітрейту камери (орієнтовно ~1 ГБ на 10 хвилин при стандартних налаштуваннях).

## Вирішення проблем

**Немає відео:**

1. ✅ Знята кришка об'єктива
2. ✅ `gs.key` на телефоні збігається з ключем камери
3. ✅ Канал і ширина каналу правильні (161 / 20 МГц)
4. ✅ Перезапустіть PixelPilot
5. ✅ Перезавантажте камеру

**Адаптер не визначається:**

- Перевірте, що телефон підтримує OTG (додаток «USB OTG Checker»)
- RTL8812EU2 без живленого хаба може не стартувати — йому потрібно стабільне 5V
- Спробуйте інший OTG-кабель

**Відео з фризами або великою затримкою:**

- Слабкий процесор телефона — спробуйте зменшити роздільну здатність/бітрейт на камері
- Перевантажений канал — змініть канал на вільніший
- Увімкніть [Adaptive-Link](/configuration/adaptive-link) для автоматичного керування бітрейтом
