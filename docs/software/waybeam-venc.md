# Waybeam — Відеоенкодер для FPV

**Waybeam** — це автономний H.265 (HEVC) відеоенкодер і RTP-стрімер для камер на чіпах SigmaStar Infinity6E (Star6E) та Infinity6C (Maruko). Розроблений спеціально для FPV-дронів із мінімальною затримкою та повним керуванням через HTTP API у реальному часі.

::: info Про назву
Сам застосунок, бінарник, файл конфігурації (`/etc/waybeam.json`) та init-скрипт називаються **`waybeam`**. Стара назва `venc` збереглася лише в адресі репозиторію `waybeam_venc` — заради стабільності URL.
:::

::: info Репозиторій
[https://github.com/OpenIPC/waybeam_venc](https://github.com/OpenIPC/waybeam_venc)
:::

### Можливості

| Функція | Опис |
| :--- | :--- |
| **Кодек** | H.265 (HEVC) з режимами CBR / VBR / AVBR / FIXQP. Кодек жорстко зафіксований — поля `video0.codec` більше немає |
| **Стрімінг** | RTP-пакетизація; Compact UDP-режим (raw NAL units). Транспорти: `udp://`, `unix://`, `shm://` |
| **HTTP API** | Зміна параметрів у реальному часі без перезапуску потоку |
| **Веб-панель** | Вбудований дешборд на порту 80: конфігурація, API-docs, ISP-тюнінг |
| **ISP IQ** | 62 ISP-параметри з підтримкою export/import профілів |
| **Custom 3A** | Вбудований AE та AWB; на Maruko `isp.aeEngine: "custom"` економить ~24% CPU при 120 fps |
| **ROI-кодування** | Пріоритет центру кадру для FPV |
| **Стійкість до втрат** | Resilience-пресети: intra-refresh (rolling GDR) + SVC-T пірамида посилань |
| **Framing** | Цифровий зум 1.25×–4× (обидва чіпи) + стабілізація `stab` / `stab-fill` (Kalman, тільки Star6E) |
| **Висока частота кадрів** | До 120 fps для IMX415 / IMX335 |
| **Аудіо** | Захват звуку, кодеки Opus / G.711a / G.711µ / PCM |
| **Snapshot** | Окремий MJPEG-канал — кадр через `/api/v1/snapshot.jpg` |
| **Запис на SD** | MPEG-TS (HEVC + аудіо), безпечний при відключенні живлення |
| **Gemini-режим** | Два VENC-канали: стрімінг + запис незалежно |
| **Адаптивний бітрейт запису** | Авто-зниження бітрейту запису якщо SD-карта не встигає (10%/с) |
| **IMU** | Драйвер BMI270 (обидва чіпи), вимкнений за замовчуванням — POC для телеметрії/sidecar |

::: warning Стабілізація змінилася
Гіроскопна стабілізація **GyroGlide-Lite (секція `eis`) була видалена у версії 0.8.0**. Тепер стабілізація реалізована через Kalman-фільтр у полі `video0.framing` (`stab` / `stab-fill`, тільки Star6E) і не потребує IMU. Драйвер BMI270 лишився, але як POC-споживач для телеметрії, а не для EIS.
:::

### Підтримувані чіпи

| Чіп | Бекенд | Сенсори | Статус |
| :--- | :--- | :--- | :--- |
| SigmaStar Infinity6E | Star6E | SSC30KQ, SSC338Q | ✅ Повна підтримка |
| SigmaStar Infinity6C | Maruko | SSC378QE | ✅ Підтримка (деякі функції — підмножина Star6E) |

::: info Кодек — лише H.265
Waybeam кодує тільки H.265 (HEVC) на обох чіпах. Поля `video0.codec` немає; старі конфіги з `"codec": "h264"` чи `"h265"` завантажуються без помилок, але ключ ігнорується.
:::

::: warning Що підтримує лише Star6E
- Стабілізація `stab` / `stab-fill`
- Scene-change IDR (`video0.sceneThreshold` / `sceneHoldoff`)
- Формат запису `hevc` (на Maruko — лише `ts`)
- HTTP-керування записом (`/api/v1/record/start|stop`); на Maruko запис лише через конфіг

Перевіряйте підтримку конкретного поля бекендом через `/api/v1/capabilities`.
:::

### Порівняння з Majestic

| | **Waybeam** | **Majestic** |
| :--- | :--- | :--- |
| **Призначення** | Спеціалізований FPV-стрімер | Загальна IP-камера |
| **HTTP API** | Повна, реалтаймова зміна полів | Обмежена |
| **WFB-інтеграція** | Нативна через Unix-сокет / SHM / UDP | Через UDP |
| **Запис на SD** | Gemini-режим (стрімінг + запис) | Обмежений |
| **Стійкість до втрат** | Resilience-пресети (intra-refresh + SVC-T) | Базова |
| **Стабілізація / зум** | Kalman-стабілізація (Star6E) + цифровий зум | Немає |
| **ISP-тюнінг** | 62 параметри в реальному часі | Базовий |
| **Ліцензія** | MIT (відкритий код) | Закритий |
