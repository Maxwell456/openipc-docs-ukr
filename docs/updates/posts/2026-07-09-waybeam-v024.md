# Waybeam v0.24 — mDNS, 144 fps та нові режими сенсорів

Після [червневого оновлення](/updates/posts/2026-06-12-waybeam-update) розробка не зупинялась: 4 липня репозиторій [OpenIPC/waybeam_venc](https://github.com/OpenIPC/waybeam_venc) синхронізували з форком розробника, і поточна версія — **v0.24.1**. Нижче — головні зміни з v0.17 та на що звернути увагу при оновленні.

---

### 🔹 mDNS — камера сама знаходиться в мережі

Waybeam тепер анонсує себе в локальній мережі через mDNS (v0.18):

- сервіс `_waybeam-venc._tcp.local` + коротке ім'я **`waybeam.local`**
- унікальне ім'я `waybeam-<суфікс>.local`, де суфікс — хвіст die ID чипа SigmaStar
- нова секція конфігу `discovery` (`enabled`, `serviceType`, `name`, `bareAlias`) — доступна також через HTTP API та WebUI
- повний 12-значний серійник читається з `GET /api/v1/config` → `data.device.serial`

Більше не треба шукати IP камери сканером — достатньо відкрити `http://waybeam.local`.

---

### 🔹 Поле `frameLost` видалено <Badge type="danger" text="BREAKING" />

У v0.19 SDK-стратегію frame-lost прибрано повністю: поля `video0.frameLost` більше не існує ані в конфізі, ані в API. Якщо воно лишилося у вашому `/etc/waybeam.json` — просто видаліть цей рядок. Натомість з'явилася нижня межа бітрейту **1000 kbps**.

---

### 🔹 Maruko: нове автоекспонування — мінус чверть CPU

У v0.22 на Maruko лишився єдиний режим AE: рідні AE+AWB вендора спершу сходяться, а далі окремий потік періодично викликає `CUS3A_RunOnce` з частотою `fps/3` (мінімум 30 Гц). Результат — **~24 % економії CPU** на 1080p100.

Поле `isp.aeEngine` на Maruko **прибрано** (на Star6E працює як раніше). Живе змінення `isp.gainMax` через API виправлено.

---

### 🔹 Нові режими сенсорів <Badge type="danger" text="BREAKING" />

Великий рефакторинг режимів на Maruko (v0.20–v0.23):

- **IMX415** — нові non-binned 16:9 режими на 1485 Mbps: `2952×1656@50`, `2952×1368@60`, `2112×1184@90`, `1920×1080@100`
- **IMX335** — нова лінійка «найкращий на кожен fps»: 30 / 50 / 60 / 90 / 100 fps
- вирівнювання ширини `video0.size` пом'якшено з /16 до /8
- `isp.keepAspect` тепер чесний: при невідповідності співвідношення сторін кадр анаморфно стискається, а не обрізається мовчки

::: warning Індекси sensor.mode перенумеровано
На обох сенсорах індекси режимів змінилися. Після оновлення перевірте `sensor.mode` у конфізі або лишіть `-1` (автовизначення). Актуальний список режимів вашої камери — `GET /api/v1/modes`.
:::

---

### 🔹 144 fps для мінімальної затримки

У v0.24 IMX335 отримав ультранизьколатентний режим **`1536×864@144`** (16:9), а live-ліміт `video0.fps` піднято до 144. Частота обмежена пропускною здатністю ISP, тому доступна лише в цьому спеціальному режимі.

---

### 🔹 Інші зміни

- **apfpv (WiFi-режим)**: на старті використовується unconnected UDP, RTP-payload обмежено 1400 байтами — менше фрагментації у WiFi-каналі
- **Аудіо «тільки в запис»**: від'ємне значення `audioPort` пише звук у файл, не відправляючи його в ефір
- **Dual-record виправлено** (v0.17.1): другий канал запису створюється одразу при `record.mode: dual`, бітрейт запису на SD незалежний від ефірного
- **WebUI**: власний дашборд можна підкласти в `/usr/share/www` — вбудований підміниться без перезбирання
- **Maruko**: контролі стабілізації у WebUI тепер неактивні — на цьому чипі немає IVE-блоку, стабілізація доступна лише на Star6E
